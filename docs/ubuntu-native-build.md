# Native Ubuntu Server build

This workflow builds the Alpine reference ISO on Ubuntu Server without Docker.

## Model

The build uses Ubuntu's `systemd-nspawn` to run Alpine tools inside the official Alpine minirootfs. This avoids attempting to install apk/aports directly into Ubuntu and keeps the resulting image tied to a pinned Alpine userspace.

It does not alter netplan, DHCP, TFTP, HTTP, firewall, or netboot.xyz configuration.

## Requirements

- x86_64 Ubuntu Server with sudo access;
- outbound HTTPS to Alpine repositories and Alpine GitLab;
- enough disk space for aports, APK caches, work files, and ISO output;
- KVM access recommended for fast QEMU tests.

## Checkout

```bash
git clone https://github.com/PlayOS-Foundation/playos-refdistro.git
cd playos-refdistro
git switch agent/adopt-alpine-reference-os
```

## One-time setup

```bash
bash scripts/setup-ubuntu-build-host.sh
```

The script installs only Ubuntu host utilities:

- `systemd-container` for systemd-nspawn;
- `qemu-system-x86`;
- `ovmf`;
- Git, curl, certificates, and xz support.

The Alpine minirootfs and caches live beneath `.build/` and are ignored by Git.

The downloaded minirootfs is checked against Alpine's published SHA-256 file before extraction. A version marker prevents silently overlaying a new Alpine version on an existing root.

## Build

```bash
bash scripts/build-iso-ubuntu.sh
```

Optional overrides:

```bash
PLAYOS_ALPINE_BRANCH=v3.24 PLAYOS_APORTS_BRANCH=3.24-stable PLAYOS_ARCH=x86_64 bash scripts/build-iso-ubuntu.sh
```

Output:

```text
out/*.iso
```

## Headless UEFI test

```bash
bash scripts/test-iso-qemu.sh
```

Or provide a specific image:

```bash
bash scripts/test-iso-qemu.sh out/playos-*.iso
```

The Alpine profile exposes `ttyS0` at 115200 baud. QEMU connects it to the terminal. Press `Ctrl-A X` to exit.

The wrapper uses KVM when the current account can access `/dev/kvm` and otherwise falls back to software emulation.

## Rebuilds

The Alpine minirootfs, aports checkout, abuild keys, and caches persist beneath `.build/alpine-rootfs`. ISO work data is recreated for each build.

To change Alpine versions, move the old root aside first:

```bash
mv .build/alpine-rootfs .build/alpine-rootfs.previous
bash scripts/setup-ubuntu-build-host.sh
```

This deliberate step prevents accidental destructive replacement.

## PXE

Do not copy the ISO directly into the netboot.xyz menu as if it were a generic kernel.

After the ISO boots successfully in QEMU, add a separate PXE publishing step that exports the exact kernel, initramfs, modloop, APK repository, and apkovl generated by the same build. PXE and ISO must consume the same profile and package inputs.

PXE automation is intentionally deferred until the baseline ISO passes UEFI boot.
